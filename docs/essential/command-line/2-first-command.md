# 第 2 章：打开终端，输入第一条命令

::: tip 本章目标
学会打开终端，理解终端界面的基本元素，输入你人生中的第一条命令！
:::

## 一、 🚀 打开你的终端

根据你使用的操作系统，打开终端的方式略有不同。

### 1.1. Windows 用户

Windows 有两个主要的命令行工具：**CMD** 和 **PowerShell**。

#### 1.1.1. 方法一：运行对话框（推荐新手）

1. 按下 `Win + R` 键，打开「运行」对话框
2. 输入 `cmd` 然后按回车 → 打开 CMD
3. 或者输入 `powershell` 然后按回车 → 打开 PowerShell

#### 1.1.2. 方法二：开始菜单搜索

1. 点击开始菜单（或按 `Win` 键）
2. 直接输入 `cmd` 或 `powershell`
3. 点击搜索结果中的应用

#### 1.1.3. 方法三：右键菜单

1. 按住 `Shift` 键
2. 在任意文件夹空白处右键
3. 选择「在此处打开 PowerShell 窗口」

::: note Windows Terminal
如果你使用的是 Windows 10 或 11，强烈推荐安装 **Windows Terminal**。它是微软官方出品的现代终端应用，界面更美观，功能更强大，可以在一个窗口里同时使用 CMD、PowerShell 甚至 WSL（Linux 子系统）。

在微软商店搜索 "Windows Terminal" 即可免费安装。
:::

### 1.2. macOS 用户

#### 1.2.1. 方法一：启动台

1. 点击 Dock 栏的「启动台」
2. 搜索「终端」或「Terminal」
3. 点击打开

#### 1.2.2. 方法二：Spotlight 搜索

1. 按下 `Cmd + Space` 打开 Spotlight
2. 输入 `Terminal` 或「终端」
3. 按回车打开

#### 1.2.3. 方法三：访达

1. 打开「访达」
2. 进入「应用程序」→「实用工具」
3. 双击「终端」

### 1.3. Linux 用户

::: tip 你可能已经是老手了
如果你在用 Linux，你很可能已经知道怎么打开终端。但为了完整性，还是列一下：
:::

- **Ubuntu/Debian**：按 `Ctrl + Alt + T`，或在应用菜单搜索「Terminal」
- **其他发行版**：查找「Terminal」、「Konsole」、「GNOME Terminal」等

## 二、 🎨 认识终端界面

打开终端后，你会看到一个（通常是黑色或深色背景的）窗口，里面有一些文字和一个闪烁的光标。

让我们来认识一下这个界面：

### 2.1. 提示符（Prompt）

提示符是终端告诉你「我准备好了，请输入命令」的标志。

**Windows CMD 的提示符：**
```
C:\Users\YourName>
```
- `C:\Users\YourName` 是你当前所在的目录
- `>` 表示「在这里输入命令」

**Windows PowerShell 的提示符：**
```
PS C:\Users\YourName>
```
- `PS` 表示这是 PowerShell
- 其他部分和 CMD 类似

**Mac/Linux 的提示符：**
```
username@hostname:~$
```
- `username` 是你的用户名
- `hostname` 是你电脑的名字
- `~` 表示你在「家目录」（相当于 Windows 的用户文件夹）
- `$` 表示你是普通用户（如果是 `#` 则表示管理员/root 用户）

::: warning 看到 # 要小心！
如果你的提示符以 `#` 结尾而不是 `$`，说明你正在以管理员（root）身份运行。在这种模式下，你有权限做任何事情，包括删除系统文件。请格外小心！
:::

### 2.2. 光标

闪烁的小方块或竖线就是**光标**，它告诉你下一个字符会输入在哪里。

## 三、 ⌨️ 你的第一条命令
好了，是时候输入你的第一条命令了！

### 3.1. Windows 用户

在提示符后面输入：
```cmd
echo Hello, World!
```
然后按 **Enter** 键。

你应该会看到：
```
Hello, World!
```

### 3.2. Mac/Linux 用户

在提示符后面输入：
```bash
echo "Hello, World!"
```
然后按 **Enter** 键。

你应该会看到：
```
Hello, World!
```

::: info 恭喜！
你刚刚执行了你的第一条命令！`echo` 命令的作用就是「回显」——把后面的内容原样输出。

就像你对着山谷喊话，山谷会把你的声音回传给你一样。这就是为什么这个命令叫 `echo`（回声）。
:::

## 四、 🔍 再来几条简单命令

让我们多尝试几条命令，感受一下命令行的工作方式。

### 4.1. 查看当前日期和时间

**Windows CMD/PowerShell：**
```cmd
date /t
time /t
```

**Mac/Linux：**
```bash
date
```

### 4.2. 查看当前目录

**Windows CMD：**
```cmd
cd
```

**Windows PowerShell / Mac / Linux：**
```bash
pwd
```

::: note pwd 是什么意思？
`pwd` 是 **P**rint **W**orking **D**irectory 的缩写，意思是「打印当前工作目录」。

这就是命令行的特点——很多命令都是英文单词或短语的缩写。一开始可能觉得神秘，但了解了缩写含义后就很好记了。
:::

### 4.3. 查看目录内容

**Windows CMD：**
```cmd
dir
```

**Windows PowerShell / Mac / Linux：**
```bash
ls
```

这条命令会列出当前目录下的所有文件和文件夹，就像打开文件管理器看到的内容一样。

## 五、 🎯 命令的基本结构

现在你已经输入了几条命令，让我们来总结一下命令的基本结构：

```
命令名 [选项] [参数]
```

- **命令名**：你想执行什么操作，比如 `echo`、`cd`、`ls`
- **选项**：修改命令行为的开关，通常以 `-` 或 `--` 开头
- **参数**：命令操作的对象，比如文件名、目录名、要显示的文字

### 5.3. 例子解析

```bash
ls -la /home
```

- `ls`：命令名（列出目录内容）
- `-la`：选项（`-l` 详细显示，`-a` 显示隐藏文件）
- `/home`：参数（要查看的目录）

::: tip 选项的两种写法
很多命令的选项有两种写法：
- 短格式：`-l`（一个短横线 + 一个字母）
- 长格式：`--list`（两个短横线 + 完整单词）

短格式可以组合，比如 `-l -a` 可以写成 `-la`。长格式更易读，短格式更方便输入。
:::

## 六、 🆘 当你不知道命令怎么用时

### 6.1. 万能的帮助命令

遇到不认识的命令？别慌，每个命令都自带说明书！

**Windows CMD：**
```cmd
命令名 /?
```

**Windows PowerShell：**
```powershell
Get-Help 命令名
```

**Mac/Linux：**
```bash
命令名 --help
# 或者
man 命令名
```

### 6.2. 试试看

输入以下命令查看 `echo` 的帮助信息：

**Windows CMD：**
```cmd
echo /?
```

**Mac/Linux：**
```bash
echo --help
```

::: note 关于 man 命令
`man` 是 **manual**（手册）的缩写。在 Mac/Linux 系统上，`man` 命令会打开一个详细的手册页面。

使用方法：
- 按 `空格键` 或 `Page Down` 向下翻页
- 按 `b` 或 `Page Up` 向上翻页
- 按 `q` 退出手册

这些手册非常详细，有时候甚至详细得让人头疼。但别担心，大部分时候你只需要看「概述」和「示例」部分就够了。
:::

## 七、 ⚡ 一些实用小技巧

### 7.1. 快捷键

| 快捷键 | 作用 | 适用系统 |
|--------|------|----------|
| `↑` / `↓` | 浏览历史命令 | 全部 |
| `Tab` | 自动补全 | 全部 |
| `Ctrl + C` | 终止当前命令 | 全部 |
| `Ctrl + L` 或 `clear` | 清屏 | Mac/Linux/PowerShell |
| `cls` | 清屏 | Windows CMD |

### 7.2. Tab 自动补全（超级有用！）

假设你想进入一个叫 `Documents` 的文件夹，你不需要完整输入：

```bash
cd Doc[按 Tab 键]
```

终端会自动补全为：
```bash
cd Documents/
```

如果有多个以 `Doc` 开头的文件/文件夹，按两次 Tab 会显示所有选项。

::: tip 养成使用 Tab 的习惯
Tab 补全不仅能节省打字时间，更重要的是**能帮你避免拼写错误**。拼错的命令或路径是新手最常见的问题之一，而 Tab 补全可以大大减少这种错误。
:::

### 7.3. 历史命令

按 `↑` 键可以调出之前输入过的命令，再按 `↓` 可以向前查看。

如果想查看完整的历史记录：

**Windows CMD：**
```cmd
doskey /history
```

**Windows PowerShell：**
```powershell
Get-History
```

**Mac/Linux：**
```bash
history
```

## 八、 🎮 小练习

在继续下一章之前，试试这些练习：

1. **打开终端**，观察提示符长什么样
2. **输入 `echo` 命令**，让终端显示你的名字
3. **使用 `pwd`（或 `cd`）** 查看当前目录
4. **使用 `ls`（或 `dir`）** 查看当前目录有什么文件
5. **按几次 `↑` 键**，体验历史命令功能
6. **输入一半命令然后按 Tab**，体验自动补全

::: important 练习的重要性
命令行不是靠「看会」的，而是靠「用会」的。就算你看懂了本章所有内容，如果不动手练习，下次打开终端你还是会忘记。

花 5 分钟完成上面的练习，它们会帮助你巩固今天学到的知识。
:::

## 九、 📝 本章小结

今天我们学会了：

1. **打开终端** —— Windows/Mac/Linux 各有各的方式
2. **认识终端界面** —— 提示符告诉你当前位置和用户身份
3. **命令的基本结构** —— `命令名 [选项] [参数]`
4. **几条基础命令** —— `echo`、`pwd`/`cd`、`ls`/`dir`
5. **获取帮助** —— `--help`、`/?`、`man`
6. **实用快捷键** —— Tab 补全、历史命令、Ctrl+C

::: warning 常见问题
**Q：我输入命令后提示「不是内部或外部命令」或「command not found」**

A：这意味着系统找不到这个命令。可能的原因：
1. 命令拼写错误（最常见！）
2. 该命令在你的系统上不存在
3. 命令没有添加到 PATH 环境变量（我们会在后面详细讲）

遇到这种情况，先检查拼写是否正确。
:::
